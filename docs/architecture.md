# OneTool Architecture

OneTool is organized as a layered Next.js App Router project with a registry-driven tool system.

## High-level flow

```
Route (src/app)
  -> Tool Registry (src/core)
  -> Tool Config (src/tools)
  -> Tool Wrapper (src/tools)
  -> Tool Module (src/modules)
```

## Folder responsibilities

- src/app: Next.js routing layer and layouts.
- src/components: Reusable UI components.
  - src/components/dashboard: Dashboard-only UI.
  - src/components/tools: Tool listing UI.
- src/core: Registry, loaders, and platform logic.
- src/modules: Feature implementations (UI + logic for each tool).
- src/tools: Tool configs and wrapper pages that bridge registry to modules.
- src/lib: Shared utilities (auth, db, tool registry exports).
- src/hooks: App hooks.
- src/services: API-facing services.
- src/store: Client state and providers.

## Routing patterns

- /tools -> tools dashboard (categories + search)
- /tools/[category] -> category listing from registry
- /tools/[category]/[tool] -> tool runtime route from registry

## What not to edit

- .next/ is build output generated by Next.js (manifests, bundles, dev cache).
- node_modules/ are installed dependencies.

If you add a new tool, create a new folder in src/tools and a matching module in src/modules, then register the config in src/core/tool-system/tool-loader.js.
